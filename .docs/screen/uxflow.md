# MyMeal 앱 UX/UI 화면 설계

> 음식 사진을 찍고, 컨디션을 기록하고, 내 몸에 맞는 식단을 추천받는 건강 관리 앱

---

## 1. 사용자 흐름

### 1.1 초기 진입 흐름

```
앱 실행 → 로그인 → 온보딩(3장) → 알림/시간 설정 → 홈
```

### 1.2 식사 기록 흐름

```
홈 → [+ 버튼] → 식사 유형 선택 → 카메라 → [촬영] → 분석 중 → (자동 저장) → 홈 타임라인
```

### 1.3 식후 반응 기록 흐름

```
식사 1시간 후 → [푸시 알림] → 클릭 → 반응 조사 화면 → [기록하기] → 홈
```

### 1.4 데이터 확인 흐름

```
홈 → [인사이트 탭] → 랭킹/패턴 확인
홈 → [저널 탭] → 캘린더 → 날짜 선택 → 일별 상세
```

---

## 2. 화면 상세 설계

### 2.1 로그인

| 요소 | 설명 |
|:---|:---|
| **로고** | 상단 "MyMeal" 로고 |
| **소셜 로그인** | 카카오, 네이버, 구글 로그인 버튼 (가로 배치) |
| **구분선** | "또는" 텍스트와 구분선 |
| **이메일 입력** | 이메일 입력 필드 |
| **비밀번호 입력** | 비밀번호 입력 필드 |
| **로그인 버튼** | "로그인" 버튼 (전체 너비) |
| **회원가입 링크** | "계정이 없으신가요? 회원가입" 텍스트 버튼 |

---

### 2.1-1 회원가입

| 요소 | 설명 |
|:---|:---|
| **헤더** | "회원가입" 제목 |
| **뒤로가기 버튼** | 좌측 상단 ← 아이콘 |
| **로고** | "MyMeal" 로고 + 서브타이틀 |
| **이름 입력** | 이름 입력 필드 |
| **이메일 입력** | 이메일 입력 필드 |
| **비밀번호 입력** | 비밀번호 입력 필드 (8자 이상) |
| **비밀번호 확인** | 비밀번호 확인 입력 필드 |
| **약관 동의** | [전체 동의] 체크박스 + 개별 약관들 |
|  | - 이용약관 (필수) |
|  | - 개인정보처리방침 (필수) |
|  | - 마케팅 정보 수신 (선택) |
| **회원가입 버튼** | "가입하기" 버튼 (전체 너비) |
| **로그인 링크** | "이미 계정이 있으신가요? 로그인" 텍스트 버튼 |

---

### 2.2 온보딩 슬라이드 (3장)

| 요소 | 설명 |
|:---|:---|
| **스킵 버튼** | 우측 상단 "건너뛰기" |
| **슬라이드 영역** | 이미지 + 제목 + 설명 |
| **페이지 인디케이터** | 하단 3개 점 (현재 페이지 하이라이트) |
| **시작하기 버튼** | 마지막 슬라이드에서만 "시작하기" 표시 |

**슬라이드 내용**

| 순서 | 제목 | 설명 | 이미지 |
|:---|:---|:---|:---|
| 1 | 음식 사진을 찍으세요 | 사진 한 장으로 식단을 기록해요 | 📸 |
| 2 | 컨디션을 기록하세요 | 식후 1시간 뒤 어떤 느낌인지 알려주세요 | 📋 |
| 3 | 내 몸에 맞는 식단을 추천받으세요 | 데이터가 쌓이면 개인 맞춤 식단을 추천드려요 | 🍽️ |

---

### 2.3 알림/시간 설정

| 요소 | 설명 |
|:---|:---|
| **헤더** | "시작하기 전에 설정을 완료해주세요" |
| **알림 권한 카드** | "식사 후 알림을 보내드릴게요" + [허용] 버튼 |
| **식사 시간 카드** | 아침/점심/저녁 시간 설정 (시간 선택기) |
|  | 기본값: 08:00, 12:00, 19:00 |
| **나중에 하기** | 하단 텍스트 버튼 (설정 → 설정 화면에서 가능) |
| **시작하기 버튼** | "MyMeal 시작하기" (전체 너비) |

---

### 2.4 홈

| 영역 | 요소 | 설명 |
|:---|:---|:---|
| **상단바** | 프로필 아이콘 | 좌측, 둥근 이미지 |
|  | 날짜 선택기 | 중앙, "< 2월 12일 수 >" 형태 |
|  | 알림 아이콘 | 우측, 벨 아이콘 |
| **식단 요약** | 오늘의 식단 요약 카드 | |
|  | 아침/점심/저녁/간식 아이콘 | 4개 아이콘 가로 배치, 기록 완료 시 채워짐 |
| **타임라인** | 날짜별 식사 목록 | 선택된 날짜의 식사 카드 리스트 |
|  | 식사 카드 | 사진 썸네일, 시간, 메뉴명, 반응 이모지 |
| **빈 상태** | "첫 식사를 기록해볼까요?" | coachmark로 안내 |
| **하단 네비게이션** | [홈] [인사이트] [+] [저널] | 4개 탭 |
|  | FAB 추가 버튼 | 하단 중앙 둥근 + 버튼 (확장됨) |

---

### 2.5 식사 유형 선택

| 요소 | 설명 |
|:---|:---|
| **헤더** | "어떤 식사인가요?" |
| **닫기 버튼** | 좌측 상단 X 아이콘 |
| **유형 버튼 4개** | 아침 🌅 / 점심 ☀️ / 저녁 🌙 / 간식 🍪 |
|  | 세로 배치, 각 버튼은 전체 너비 |
|  | 선택 시 하이라이트 효과 |
| **시간 힌트** | "지금은 아침 식사 시간이네요" (선택사항, 현재 시간 기반) |
| **선택 동작** | 선택 후 자동으로 카메라 화면으로 진입 |

---

### 2.6 카메라

| 요소 | 설명 |
|:---|:---|
| **헤더** | "[유형] 식사 인증하기" (선택된 유형 표시) |
| **닫기 버튼** | 좌측 상단 X 아이콘 |
| **뷰파인더** | 전체 화면 카메라 프리뷰 |
| **촬영 가이드라인** | 중앙 사각형 가이드라인 |
| **식사 유형 탭** | 하단 4개 탭 (유형 변경 가능) |
| **갤러리 버튼** | 좌측 하단 |
| **촬영 버튼** | 우측 하단, 둥근 큰 버튼 |
| **플래시 버튼** | 상단 우측 |
| **전면/후면 토글** | 하단 중앙 |

### 2.8 식후 반응 조사

| 요소 | 설명 |
|:---|:---|
| **헤더** | "식후 컨디션 체크" |
| **닫기 버튼** | 좌측 상단 X 아이콘 |
| **식사 정보** | 촬영한 식사 사진 썸네일 + 시간 |
| **슬라이더 3개** | 소화 상태, 포만감, 에너지 레벨 |

|  | 포만감: 여전히 배고픔 ~ 너무 부름 (5단계) |
|  | 에너지: 나른함 ~ 에너지 넘침 (5단계) |
| **체크리스트** | 속쓰림, 가스 찬, 배부펑, 두통 (복수 선택) |
| **나중에 하기** | 하단 "나중에 하기" 버튼 |
| **기록하기 버튼** | 하단 "기록하기" (전체 너비) |

|  | 진행률 바 표시 |

## 5. 상태 관리

### 5.1 로딩 상태

| 상황 | UI |
|:---|:---|
| AI 분석 중 | 스피너 + "[유형] 메뉴를 분석 중이에요..." |
| 저장 완료 | 체크 애니메이션 ✅ + "저장되었어요!" |

### 5.2 빈 상태

| 상황 | 메시지 |
|:---|:---|
| 첫 식사 전 | "첫 식사를 기록해볼까요?" |
| 캘린더 빈 날 | "이 날은 기록이 없어요" |
| 인사이트 빈 상태 | "식사 3건 더 기록하면 첫 인사이트가 생성돼요" |

### 5.3 에러 상태

| 상황 | 해결책 |
|:---|:---|
| AI 분석 실패 | "분석에 실패했어요. 다시 시도해볼까요?" + 재촬영 버튼 |
| 네트워크 오류 | "네트워크 연결을 확인해주세요" + 재시도 버튼 |
